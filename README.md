# BVOLT Infrastructure (bvolt-infra)

Infrastructure stack for **BVOLT (Battery Virtualization and
Optimization Layer for Telemetry)**.

This repository contains the **database and visualization
infrastructure** used by BVOLT to store and analyze telemetry data from
microgrid components such as:

-   photovoltaic arrays
-   battery banks
-   inverters
-   hydrogen systems
-   laboratory experimental platforms

The stack is designed to run **inside the laboratory network**, while
still allowing **secure remote access for researchers**.

------------------------------------------------------------------------

# Architecture

BVOLT follows a modular architecture composed of three repositories:

    bvolt-core
        │
        │ telemetry writes
        ▼
    bvolt-infra
        │
        ├── InfluxDB (time-series database)
        └── Grafana (visualization)

Telemetry pipeline:

    Microgrid hardware
           │
           ▼
    Microgrid integration services
           │
           ▼
    BVOLT Core
           │
           ▼
    InfluxDB
           │
           ▼
    Grafana dashboards

------------------------------------------------------------------------

# Services

## InfluxDB

InfluxDB is used as the **time-series database** for storing telemetry
generated by microgrid components.

Typical measurements include:

-   voltage
-   current
-   power
-   frequency
-   temperature
-   state of charge

Default access endpoint:

    http://<server-ip>:8086

------------------------------------------------------------------------

## Grafana

Grafana provides **visualization dashboards** for telemetry data stored
in InfluxDB.

Dashboards can display:

-   PV production
-   battery SOC
-   inverter power flows
-   grid import/export
-   hydrogen system telemetry

Default access endpoint:

    http://<server-ip>:3000

# Secure Remote Access Using Tailscale

The laboratory infrastructure runs **inside the university network**,
which usually blocks direct inbound connections from the internet.

To allow researchers to access the dashboards remotely without exposing
the services publicly, BVOLT uses **Tailscale**.

Tailscale is a **mesh VPN built on WireGuard** that allows devices to
communicate securely over a private network.

## Architecture

    Researcher Laptop
           │
           │ Tailscale VPN
           ▼
    Lab Server
       ├── Grafana :3000
       └── InfluxDB :8086

Each authorized device joins the same **Tailnet**, enabling access to
the infrastructure as if it were on the same local network.

------------------------------------------------------------------------

# Installing Tailscale on the Server

Install Tailscale on the laboratory machine:

    curl -fsSL https://tailscale.com/install.sh | sh
    sudo tailscale up

Follow the authentication link printed in the terminal.

Retrieve the assigned VPN IP:

    tailscale ip -4

Example:

    100.84.22.19

------------------------------------------------------------------------

# Accessing BVOLT Infrastructure via Tailscale

Once connected to the same Tailnet, services can be accessed using the
Tailscale IP.

Example:

Grafana:

    http://100.84.22.19:3000

InfluxDB:

    http://100.84.22.19:8086

This approach:

-   avoids exposing services to the internet
-   works behind university NAT/firewalls
-   allows access from any authorized device

------------------------------------------------------------------------

# Recommended Access Policy

For security reasons:

-   **Grafana** should be accessible to all team members who need
    dashboards
-   **InfluxDB** should be accessed only by backend services or advanced
    users
-   access tokens should be restricted by role

Example token usage:

Token        Purpose
------------ -------------------------
`BVOLT Core`   telemetry write

`Grafana`      read-only dashboards

`Admin`        database administration

------------------------------------------------------------------------

# Backup Strategy

InfluxDB telemetry data should be backed up regularly.

Example manual backup:

    docker exec bvolt-influxdb influx backup /backup

Backups should be stored in:

-   NAS storage
-   institutional storage
-   cloud object storage
